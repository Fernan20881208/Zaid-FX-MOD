# Zaid-FX-MOD

Android-first post-processing and visual-filter framework for Geometry Dash using Geode.

> **v0.2.0 testing build:** the complete game frame is captured and drawn through a live GLSL post-processing pipeline. Geode sliders update the shader immediately while the game is running.

## Download and installation

Download the latest `.geode` package from the [GitHub Releases page](https://github.com/Fernan20881208/Zaid-FX-MOD/releases/latest), then import or place it in the Geode mods directory on Android.

Current target:

- Geometry Dash Android `2.2081`
- Geode `5.7.1`
- Android ARM64 (`Android64` / `arm64-v8a`)

## Live visual controls

The following controls are connected directly to uniforms in `color-grade.frag` and are sent on every processed frame:

- Effect intensity: `u_intensity`
- Brightness: `u_brightness`
- Exposure: `u_exposure`
- Contrast: `u_contrast`
- Saturation: `u_saturation`
- Gamma: `u_gamma`
- Vignette: `u_vignette`
- Sharpen: `u_sharpen`
- Pixel size for sharpening: `u_texelSize`

Saturation uses `0` for grayscale, `1` for the original color level and values above `1` for stronger colors. Effect intensity uses a normalized `0.0` to `1.0` range.

Presets write their values into the real Geode settings. Moving an individual slider changes the selected preset to **Custom**, preventing a separate preset copy from replacing the slider value.

## Diagnostics

`Shader debug logging` is enabled by default in this testing build. Logs include:

- The value generated by every slider.
- The sanitized value received by the renderer.
- The uniform name and value sent to OpenGL.
- The final settings revision used for a rendered frame.
- Shader compilation, missing-uniform and pipeline creation errors.

Disable this option after device testing to reduce log volume.

## Visual effects

Currently implemented:

- Global effect-intensity blend.
- Brightness and exposure.
- Contrast and saturation.
- Gamma correction.
- Fullscreen vignette.
- Neighbor-sampling sharpen filter.
- Clean, OLED, Vibrant, Cinematic, Competitive and **RTX Fake** presets.

Planned next:

- Quarter-resolution bloom.
- Tone mapping and light shafts.
- Dynamic quality based on frame time.
- Android context-loss restoration validation.

**RTX Fake** is a stylized combination of post-processing effects. It is not hardware ray tracing.

## Build from source

Install the Geode CLI, SDK binaries and Android NDK, then run:

```bash
geode sdk install-binaries -p android64
geode build -p android64
```

The package should be produced under `build-android64`.

## Repository structure

- `src/rendering/PostProcessRenderer.*`: global frame capture, shared settings state and per-frame uniforms.
- `src/rendering/ShaderProgram.*`: GLSL compilation and cached uniform access.
- `src/settings`: typed and sanitized access to Geode settings.
- `resources/shaders`: packaged GLSL ES shaders.
- `.github/workflows`: verified Android64 build and versioned release workflow.
- `docs/ROADMAP.md`: implementation milestones and validation gates.

## Safety and compatibility goals

- Never modify gameplay physics or hitboxes.
- Restore OpenGL resources after Android context loss.
- Avoid overriding Geometry Dash shader triggers.
- Allow effects to be disabled immediately.
- Fall back to normal game rendering if pipeline creation fails.
- Prefer reduced-resolution multipass effects on mobile GPUs.
